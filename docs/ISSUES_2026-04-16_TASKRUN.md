# TaskRun 遗留问题记录（2026-04-16）

## 背景
- 页面：`任务执行`（TaskRun）
- 记录时间：2026-04-16
- 当前决定：仅记录问题，不在本次继续修复

## 现象 1：已切换到新任务，时间线仍显示之前任务的照片
- 期望：切换/启动新任务后，时间线只展示当前 `Task ID` 对应的轮次图片。
- 实际：页面中仍可看到历史任务（之前任务）的轮次图片卡片。
- 影响：用户难以判断当前任务结果，存在误判风险。

## 现象 2：轮次图片无法显示（破图）
- 期望：时间线卡片应正常展示每轮生成图预览。
- 实际：卡片中图片区域为破图（`round-1/round-2/...` 均不显示）。
- 影响：无法直观看到每轮生成结果，排错效率下降。

## 现象 3：右侧上下文状态栏数值异常
- 期望：`占用率`、`Tokens` 显示有效数值（例如 `xx.x%`、`a/b`）。
- 实际：长期显示 `--%`、`--`。
- 影响：无法判断上下文消耗情况，压缩策略状态不可观测。

## 现场截图要点
- 时间线中可见多轮卡片（第 1~5 轮）但均为破图。
- 右侧状态区显示：
  - `状态: FAILED`
  - `重试次数: 5`
  - `累计费用: $0.1815`
  - `上下文状态（简洁模式）` 下 `占用率` 与 `Tokens` 为 `--`。

## 初步排查方向（留待明天）
- 任务切换与状态清理：
  - 检查 `activeTaskId` 切换时 `roundPreviews` 的清空/覆盖逻辑。
  - 检查是否存在“历史 artifacts hydrate 覆盖当前任务视图”的时序问题。
- 预览图链路：
  - 检查 `generated_image_path / preview_image_path` 的真实文件是否存在。
  - 检查数据库记录路径与实际文件后缀/编码一致性。
  - 检查 `toFileUrl(...)` 与渲染侧 `<img src=...>` 的路径兼容性。
- 上下文数值链路：
  - 检查 `q.getContextUsage()` 失败时的兜底是否被正确触发。
  - 检查 `LoopEvent.contextUsage` 是否在主进程产生后传到 renderer store。
  - 检查 store 是否被旧事件覆盖导致最终显示为 `--`。

## 验收标准（修复时参考）
- 新任务启动后，时间线不再混入旧任务图片。
- 当前任务至少一轮图片可在时间线正常显示。
- 右侧 `占用率` 与 `Tokens` 显示有效数值（非 `--`）。
